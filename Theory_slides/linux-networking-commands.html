Linux Networking Reference

net-tools vs iproute2

Comparative command reference for MAC/IP address and route management

⚠ net-tools is deprecated — prefer iproute2
Task net-tools (old) iproute2 (modern) Notes
// Interfaces & Link Layer
Show all interfaces ifconfig -a ip link show ip link shows L2 info; ip addr show adds IP addresses
Bring interface up ifconfig eth0 up ip link set eth0 up
Bring interface down ifconfig eth0 down ip link set eth0 down
Change MAC address ifconfig eth0 hw ether AA:BB:CC:DD:EE:FF ip link set eth0 address AA:BB:CC:DD:EE:FF Interface must be down first
Set MTU ifconfig eth0 mtu 9000 ip link set eth0 mtu 9000 Jumbo frames for high-throughput networks
Enable promiscuous mode ifconfig eth0 promisc ip link set eth0 promisc on Needed for packet capture / bridging
// IP Addresses
Show IP addresses ifconfig eth0 ip addr show eth0 split ip separates L2 (link) from L3 (addr)
Assign IP address ifconfig eth0 192.168.1.10 netmask 255.255.255.0 ip addr add 192.168.1.10/24 dev eth0 ip uses CIDR notation; does not replace existing address
Remove IP address ifconfig eth0 0.0.0.0 ip addr del 192.168.1.10/24 dev eth0 Must specify the exact address to remove
Add secondary IP (alias) ifconfig eth0:1 10.0.0.1 netmask 255.0.0.0 up ip addr add 10.0.0.1/8 dev eth0 iproute2 natively supports multiple addresses; no alias needed
Assign IP and bring up ifconfig eth0 192.168.1.10 netmask 255.255.255.0 up ip addr add 192.168.1.10/24 dev eth0
ip link set eth0 up		 split two commands in iproute2
// Routing
Show routing table route -n ip route show -n avoids DNS lookups in old command
Add default gateway route add default gw 192.168.1.1 ip route add default via 192.168.1.1 "via" maps clearly to "next hop" concept
Delete default gateway route del default gw 192.168.1.1 ip route del default via 192.168.1.1
Add static route route add -net 10.0.0.0 netmask 255.0.0.0 gw 192.168.1.1 ip route add 10.0.0.0/8 via 192.168.1.1 CIDR is shorter and cleaner
Add route via interface route add -net 10.0.0.0 netmask 255.0.0.0 dev eth0 ip route add 10.0.0.0/8 dev eth0 For directly connected networks
Delete static route route del -net 10.0.0.0 netmask 255.0.0.0 ip route del 10.0.0.0/8
Show route for destination route get 8.8.8.8 (not available) ip route get 8.8.8.8 iproute2 exclusive — shows exact path used
// ARP — Address Resolution Protocol
Show ARP table arp -n ip neigh show "neigh" = neighbor table (ARP for IPv4, NDP for IPv6)
Add static ARP entry arp -s 192.168.1.5 AA:BB:CC:DD:EE:FF ip neigh add 192.168.1.5 lladdr AA:BB:CC:DD:EE:FF dev eth0 nud permanent nud = Neighbor Unreachability Detection state
Delete ARP entry arp -d 192.168.1.5 ip neigh del 192.168.1.5 dev eth0
Flush ARP cache (not available) ip neigh flush dev eth0 iproute2 exclusive
// Socket & Connection Monitoring
Show listening ports netstat -tuln ss -tuln same flags
Show connections with process netstat -tunp ss -tunp ss is faster via netlink; shows more TCP detail
Show socket summary netstat -s ss -s
net-tools package: ifconfig · route · arp · netstat  |  iproute2 package: ip · ss
